# First Message · Format and Hard Rules

## The three parts of the message (in this order, the order does not change)
1. **What you noticed:** a SPECIFIC detail from that brand's ad. The brand name + a real observation from the ad. This part proves the message is tailored; if it is generic, the message is spam.
2. **What you suggest:** a single concrete idea tied to the weak point. Do not pitch your service, say the idea. The idea is given for free; the work is implementing it.
3. **One clear question:** a question that is easy to answer, with a yes/no or a single sentence. One. No meeting push.

## Length
4-6 lines. It fits on the screen in a DM. A long message does not get read.

## Structure example (a skeleton, not copied word for word)
```text
hi <brand>, <a specific observation from the ad: which ad, how long it has been running, what caught your eye>.

<the weak point, as an observation and not an accusation> <one concrete idea>.

<one clear question>
```

## Hard rules
- **A specific detail is required.** Not "I saw your ads", but "in that ad of yours that has been running for 6 months". The detail comes from the `prospects/` file; if it is not there, the message is not written, and you say "not enough information for this prospect".
- **Frame the weak point as an observation, not an insult.** Bad: "your ad is terrible, your hook is awful." Good: "the first sentence opens with the brand name, and openings like that usually get scrolled past in the first second."
- **One idea.** Do not list five suggestions. Give the single strongest idea.
- **One question.** There is one question mark at the end of the message. If there are two, rewrite it.
- **No selling.** Price, packages, "our services", and a calendar link do not appear in the first message. The job of the first message is to get a reply.
- NO made-up numbers, fake references, or non-existent work examples. If it is not in `brand/01-service.md`, leave a `[put your own example/number here]` placeholder.
- Do not invent information about the prospect that is not written in the `prospects/` file (revenue, team, founding year, which agency they work with).
- The banned words in `brand/02-voice.md` apply. Bulk-send language is banned.
- No em dashes. Separators: period, comma, colon, middle dot (·).

## Under each message (for the user to see, not part of the message)
- `basis:` which observation came from which ad.
- `weak point:` which of the four.
- `why they might reply:` one line.
