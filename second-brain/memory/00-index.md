# 00 · Index (map of the brain)

> This file tells you what is in the brain and which topic lives in which file.
> Claude Code reads it before every task, then pulls the right file for the job.
> When you add a new file, add a line here. If the index is wrong, the loading is wrong.

## Files and what they hold

| File | What is in it | When it is read |
|---|---|---|
| `01-core.md` | who, what we do, for whom, current priority, what we are not | always |
| `02-voice.md` | how we write, address, banned words | when writing text |
| `03-projects.md` | active projects, their status, next step | on project work |
| `04-people.md` | customers, partners, who for what | on people-related work |
| `05-decisions.md` | accumulated decisions, "do not touch now", what is featured | on decisions + output |
| `06-resources.md` | tools, links, templates, recurring info | on tool/resource work |

## Quick lookup (if what you need is here, know the file)
- "what did we decide last time" -> `05-decisions.md`
- "what is the status of that project" -> `03-projects.md`
- "who is that customer, what did we talk about" -> `04-people.md`
- "where is that tool's link / password" -> `06-resources.md` (do NOT store passwords, only where they are)
- "how should I write this" -> `02-voice.md`

## Note
As the brain grows, if one topic no longer fits a single file, split it (e.g. `03-projects.md` -> `03a-active.md`, `03b-archive.md`) and note it here. Every new file goes into this table, otherwise the system cannot find it.
