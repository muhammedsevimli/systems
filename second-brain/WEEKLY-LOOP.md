# Weekly Loop · 5 minutes

> This is the only thing that keeps your second brain alive. Once a week, 5 minutes.
> Skip it and the brain goes stale, loads the wrong context. Do it and it grows with you.

## Ritual (same day every week, e.g. Friday afternoon)

1. What happened this week?
   A new decision, a test that worked or flopped, something from a customer.
   Write it at the top of `memory/05-decisions.md` with today's date.

2. Are the projects current?
   Update any project whose status changed in `memory/03-projects.md`. Move the finished ones to the archive.

3. Did a new person / resource come in?
   New customer -> `memory/04-people.md`. New tool/template -> `memory/06-resources.md`.

4. Did the priority change?
   Are `memory/01-core.md` "current priority" and `05-decisions.md` "what we are featuring" still correct.

5. Is the index correct?
   If you added a new file, did you add a line to the `memory/00-index.md` map. If not, the system will not load that file.

## Monthly (once a month, 15 minutes)
- Does `memory/01-core.md` still describe you.
- If `memory/05-decisions.md` has grown, trim the stale ones to the archive.
- If a file has grown too big, split it and note it in the index.
