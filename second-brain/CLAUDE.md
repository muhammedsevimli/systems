# Second Brain · Auto Context-Loading Rule

> This file is read automatically at the START of every Claude Code session you open in this folder.
> You do nothing. Claude Code loads it on its own while working in this folder.
> The goal: never explain your work's memory (projects, decisions, resources, people) from scratch again.

## Before every task, ALWAYS read these two files
1. `memory/00-index.md`: what is in the brain, which file holds what. This is your map.
2. `memory/01-core.md`: who, what we do, what the priority is now. Needed on every task.

These two files are small and always read. Load the rest by task (table below).

## Load context by task (not everything, only what is needed)
As the brain grows, pull the files that fit the job instead of reading everything. `00-index.md` tells you which file holds which topic. The rule:

| Task type | Also read |
|---|---|
| Writing content / email / proposal | `memory/02-voice.md`, `memory/05-decisions.md` |
| Working on a project | `memory/03-projects.md`, `05-decisions.md` for that project |
| Customer / people-related work | `memory/04-people.md` |
| Using a tool / resource | `memory/06-resources.md` |
| Making a decision / "what did we say last time" | `memory/05-decisions.md` |

If unsure, check the mapping in `00-index.md`. Do not load unnecessary files, keep the context clean.

## Output rules
- When you start a task, say in one line which files you read (e.g. "core + projects + decisions read"). This makes it clear you are working from memory.
- Never contradict the "what we are NOT" lines in `01-core.md`.
- Do not touch the topics in the "do NOT touch now" list in `05-decisions.md`.
- If you are producing text, follow the tone in `02-voice.md`, do not use banned words.
- Do not make things up. If it is not in the brain, say "this is not in the brain, want me to add it".

## Write back new information (write-back)
If a new fact, decision, person, or resource becomes clear while working, record it in the right file:
- new decision -> top of `05-decisions.md` with the date.
- new project / project status change -> `03-projects.md`.
- new person / customer / partner -> `04-people.md`.
- new tool / link / template -> `06-resources.md`.
- if you added a new file, add a line to the `00-index.md` map.

After every write-back, give a one-line summary: "added this to decisions". This way the brain grows every session.

## Weekly loop
Every week, run the 5-minute ritual in `WEEKLY-LOOP.md`. The brain stays current, stale info moves to the archive, and the index stays correct.
